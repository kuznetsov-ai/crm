5 Правила применения — обязательные паттерны

Фон страниц

/* ✅ Всегда — страница */ .page-wrapper { background: var(--bg); /* #fff light / #1a191e dark */ } /* ❌ Никогда для страниц */ .page-wrapper { background: var(--surface); /* даёт светлую полосу в тёмной теме */ } /* Next.js — в layout или page.tsx */ <div className="min-h-screen bg-[var(--bg)]"> /* ✅ */ <div className="min-h-screen bg-[var(--surface)]"> /* ❌ */

Скилл-теги (навыки)

/* ✅ Оранжевые — видны на любом фоне */ <span className="bg-[var(--accent-soft)] text-[var(--accent)] rounded px-1.5 py-0.5 text-[10px]"> /* ❌ Серые — невидимы в тёмной теме */ <span className="bg-[var(--surface-2)] text-[var(--text-subtle)]">
BPMN Agile Kafka PostgreSQL +22

Делители таблиц

/* ✅ CSS var — адаптируется к теме */ <tbody className="divide-y divide-[var(--border)]"> /* ❌ Хардкод — белые полосы в тёмной теме */ <tbody className="divide-y divide-gray-200">

Цвета светлых фонов (badges, pills)

/* ❌ Любые *-50 цвета — светлые в тёмной теме */ bg-blue-50 → bg-[var(--info-soft)] bg-green-50 → bg-[var(--success-soft)] bg-orange-50→ bg-[var(--accent-soft)] bg-red-50 → bg-[var(--danger-soft)] bg-yellow-50→ bg-[var(--warning-soft)]

Переключение тем — SSR без flash

/* layout.tsx — обязательно для Next.js */ <html suppressHydrationWarning> /* ← без этого будет hydration error */ <head> <script dangerouslySetInnerHTML={{ __html: themeInitScript }} /> </head> <body style={{ background: 'var(--bg)', color: 'var(--text)' }}> /* inline style вместо Tailwind — реагирует на runtime theme change */ </body> </html>

StatCard — стиль карточки

/* Всегда совпадает с Card компонентом */ background: var(--bg); /* НЕ surface — иначе темнее соседних карточек */ border-radius: 0.5rem; /* совпадает с rounded-lg в Card */ border: 1px solid var(--border);
